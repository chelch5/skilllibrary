# Purpose
Structures deities, factions, history, settings, and lore consistency for world-heavy creative projects.

# When to use this skill
Use when:
- Building a consistent fictional world with factions and history
- Maintaining lore consistency across game content
- Designing systems for lore-driven gameplay (reputation, factions)

Do NOT use when:
- Pure mechanical game design without narrative
- Technical game systems without worldbuilding components

# Operating procedure
1. Define world pillars (geography, factions, history)
2. Create lore canon document with timeline
3. Build faction relationship matrix
4. Link lore facts to gameplay consequences
5. Review new content against lore constraints

# Output defaults
World bible documents, faction maps, lore canon files, timeline spreadsheets

# Failure handling
If contradictions arise, flag them explicitly. Maintain a known-issues list for later revision.
