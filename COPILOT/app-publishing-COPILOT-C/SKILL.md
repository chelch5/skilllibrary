# Purpose
Guides the process of publishing apps to the App Store, Google Play, or other distribution platforms.

# When to use this skill
Use when:
- Preparing an app for App Store or Google Play submission
- Setting up app store metadata, screenshots, and descriptions
- Managing app versioning and phased rollout

Do NOT use when:
- Backend-only services without app store submission
- Web apps deployed via standard hosting

# Operating procedure
1. Review store guidelines and submission requirements
2. Prepare metadata (description, keywords, screenshots)
3. Set version number and build number correctly
4. Submit for review and monitor review status
5. Plan phased rollout or staged release

# Output defaults
App store submission checklist, metadata files, release notes, review response templates

# Failure handling
If review is rejected, read rejection reason carefully. Respond factually with evidence.
